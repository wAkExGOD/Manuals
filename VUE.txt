1 (10).
- data() - функция, возвращающая объект, описывающий состояние компонента
- состояние - то, что меняется с течением времени
- двустороннее связывание с v-model="var1"
- подписываться на события через v-on: (= @) !(доступны только переменные, объявленные в компоненте - data, methods)
__ `this` внутри методов указывает на экземпляр Vue (можно обратиться к переменной name в data - this.name)
- вывод данных в шаблон (интерполяция) через {{ }}
- две ключевые структурные директивы - v-for="", v-if="" !(их вместе использовать в одном элементе нельзя)
__ v-show="" - отображает/нет, только меняя свойство display (= элемент остается в DOM'е)

2 (13).
- компоненты - для разделения системы на части и дать возможность описать интерфейсы этих частей
- интерфейс - это соглашение, по которому элементы системы (=компоненты) взайимодействуют между собой
- каждый компонент описывает:
  1) что у него на входе
  2) что у него на выходе

5 (22).
- refs (рефы) нужны только тогда, когда вы не можете сделать что-то или получить что-то средствами Vue
- refs доступны только после того, как комопнент смонтирован (В created не доступны)
- refs не реактивны (Не стоит их использовать в computed)
- refs лучше использовать как можно меньше
